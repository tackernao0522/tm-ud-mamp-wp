```html:sample.html

 	<li>02></li>
</ul>

```

+ オプションで`ビジュアルエディター`の戻ることができるがクラッシックというブロックに入っていくようになる<br>

+ 下のブロックを追加して `ブロックエディターで編集したブロックです`を入力してみる<br>

```html:sample.html
<!-- コードエディターで編集した状態 -->
<h2>見出し</h2>
<ul>
<li>箇条書き</li>
<li>02&gt;</li>
</ul>
<p>本文です</p>
<!-- ここまでコードエディターで編集した状態 -->

<!-- ここからブロックエディタで編集した状態 -->
<!-- wp:paragraph --> <!-- このコメントアウトはいじってはだめ-->
<p>ブロックエディターで編集したブロックです</p>
<!-- /wp:paragraph --> <!-- このコメントアウトはいじってはだめ-->
<!-- ここまでブロックエディタで編集した状態 -->
```

+ `コードエディターを終了`でコードエディタとブロックエディタで同居することになる<br>

条件分岐タグとの使用
アーカイブページ (is_archive() で判定) またはカテゴリーページ (is_category() で判定) 表示時に、the_content() タグの替わりに the_excerpt() を用いて抜粋文を表示する例です。

function custom_excerpt_length( $length ) {
     return 20;	
}	
add_filter( 'excerpt_length', 'custom_excerpt_length', 999 );
フィルターで [...] 文字列の変更・削除
WordPress version 2.9では概要の[...]文字列を変更するための新たなフィルターが追加されました。下記メソッドは適切なWordPress versionに対して使用してください。両方を同時に使用しないでください。

バージョン 2.9 以降のみ

function new_excerpt_more($more) {
	return '[.....]';
}
add_filter('excerpt_more', 'new_excerpt_more');
バージョン 2.8.x 以前の場合
function new_excerpt_more($excerpt) {
	return str_replace('[...]', '...', $excerpt);
}
add_filter('wp_trim_excerpt', 'new_excerpt_more');

## 101. ナビゲーションメニューを作ろう

1. 投稿の中に固定ページのリンクを貼る方法

+ wp管理画面 => `投稿` => `ブログを開設しました` => `編集` => `ブロックエディター` => `このブログについて`と入力してみる => `リンクマーク`とクリック => `検索またはURLを入力` => `このブログに...`と入力してみる => `このブログについて`を選択 => `更新` => `投稿を表示` => この投稿ページにリンクされる<br>

2. グローバルナビゲーションの作成の仕方

+ wp管理画面 => `外観` => `メニュー` => `メニュー名` => `グローバルナビ`と入力してみる => `メニューを作成` => `固定ページ` => `最近 or すべて表示` => `このブログについて`にチェック => `メニューに追加` => 下記へ続く<br>

+ `メニュー項目を追加` => `カテゴリー` => `お知らせ`にチェック => `メニューに追加` => `メニュー設定` => `メニューの位置`(twenty-twenty-oneの場合はメインメニューとサブメニューがある) => `メインメニュー`にチェック => `メニューを保存` => 下記へ続く<br>

+ 新しいタブでサイトを開く グローバルナビが生成されている<br>

＊ `お知らせ`を前にもっていきたい場合<br>

+ `メニュー` => `メニュー構造` => `お知らせ`をマウスで掴んで上に持っていくことができる => `メニューを保存` => サイトを更新してみる<br>

+ `メニュー` => `メニュー項目を追加` => `カスタムリング` => `URL` => `/blog/`と入力してみる => `リンク文字列` => `ホーム`と入力してみる => `メニューに追加` => `メニュー構造` => `ホーム`を一番上にもっていく => 下記へ続く<br>

+ `メニューを保存` => サイトを更新してみる ホーム(Topページ)へのメニューができている<br>

+ + `メニュー` => `メニュー項目を追加` => `カテゴリー` => `ニュースリリース`にチェック => `メニューに追加` => ニュースリリースはお知らせカテゴリーの子なので`メニュー構造` => `お知らせ`の下に移動して少し右にずらして`ニュースリリース`を置く => 下記へ続く<br>

+ `メニューを保存` => サイトを更新してみる<br>

+ `メニュー` => `メニュー構造` => `メニュー構造` => `メニュー設定` => `サブメニュー`にもチェックを入れてみる => `メニューを保存` => サイトを更新してみる(画面の下部にメニューが作成されている)<br>

* サブメニューはメインメニューとは違うメニューを作る場合

+ `メニュー` => `新しいメニューを作成しましょう` => `メニュー構造` => `メニュー名` => `メニュー設定` => `サブメニュー` => `メニューを作成` => `メニュー項目を追加` => `固定ページ` => `自己紹介` => `メニューに追加` => `メニューを保存`<br>

+ サイトを更新(サブメニューはメインメニューとは違うリンクになる)<br>

+ `メニュー` => `位置を管理` => `サブメニュー` => `指定されたメニュー` => `グローバルナビ`を選択` => `変更を保存`<br>

+ サイトを更新(サブメニューはメインナビと同じメニューになる)<br>

## 102. 投稿のステータス「下書き」「非公開」「パスワード保護」

+ wp管理画面 => `投稿` => `新規追加` => `タイトルを追加` => `下書き`と入力してみる => `下書き保存` => 新しいタブでTOPページを開いてみる(下書きにした記事は表示されていない。プレビューはできる) <br>

+ `⚙マーク`=> `投稿` => `表示状態` => `公開` => `非公開(完成した記事を一般には見せないという概念になる)` <br>

+ 上記のタイトルを分かりづらいので`非公開の記事`に変更しておく<br>

+ `更新` => TOPページを再読み込みしてみる<br>

+ 上記の記事の編集画面 => タイトルを`ステータスの練習`と変更しておく => `⚙マーク` => `投稿` => `表示状態` => `パスワード保護` => パスワードを設定してみる => `更新` => この記事の詳細画面を見ようとするとパスワード入力欄が出てパスワードを入れると遷移して表示される<br>

## 103. 投稿日時と変更と予約投稿

+ wp管理画面 => `投稿` => `新規追加` => `タイトルを追加` => `日付について`と入力してみる => `公開` => `公開` => `投稿を表示`<br>

+ 日付日時は登校日の日付になっている<br>

+ 上記の記事の編集画面 => `⚙` => `公開` => 実際に公開した日時が表示されている<br>

+ `投稿` => `新規追加` => `タイトルを追加` => `本当は過去の記事`と入力してみる => `公開` => `公開` => `投稿を表示` <br>

+ 本当は過去の日付にしたい 投稿を編集 => `⚙` => `公開` => 日付のところをクリック => `実際に投稿したい日時を選択する` => `更新` => サイトを表示してみる<br>

+ wp管理画面 => `投稿` => `新規追加` => `タイトルを追加` => `未来の記事`と入力してみる => 公開をする前に `⚙` => `公開` => `今すぐのところをクリック` => `未来の日付に設定してみる` => `予約...` => `予約投稿`<br>

+ TOPページに行くと上記の未来の記事は表示されない。予約した日時が到来してしばらくすると公開されるようになる(最初の訪問者がアクセスしてきたときに公開される仕組み)<br>

+ 不要なプラグインを削除してみる => `Hello Dolly` => `削除` => `Akismet Anti-Spam` => `削除`<br>

## 105. SEO SIMPLE PACK で SEO設定

+ wp管理画面 => `SEO PACK` => `一般設定` => `基本設定` => `区切り文字` => `-`を選択してみる => `設定を保存する` (タブタイトルが `たかちゃんブログ - ... となる) ＊何と選んでも構わない<br>

+ `たかちゃんブログ - ...`の `...`の部分が不要であれば `「フロントページ」のタイトルタグ`の部分を編集すればよい => `%_site_title_% %_sep_% %_tagline_%` を `%_site_title_%`のようにしてみる => `設定を保存する`<br>

+ サイトをリロードすると `たかちゃんブログ`だけになっている<br>

+ wp管理画面 => `SEO PACK` => `一般設定` => `基本設定` => 「フロントページ」のディスクリプション => `たかちゃん代表のブログサイト`を入力してみる(下記へ続く) <br>

+ `「フロントページ」のキーワード` は近年でmetatagだが機能しないと言われているので書かなくてもよいと思われる(下記へ続く)<br>

+ `各投稿のキーワード設定が空の時、上記と同じキーワードを出力する`にチェックを入れておく(下記へ続く)<br>

+ `特殊ページ設定`の部分も必要に応じて編集すればよい (下記へ続く)<br>

+ `投稿ページ`(SEO PACK) => `上記と同じように必要に応じて編集すればよい`(下記へ続く)<br>

+ `タクソノミーアーカイブ` => `上記と同じように必要に応じて編集すればよい`(下記へ続く)<br>

+ `その他アーカイブ` => `上記と同じように必要に応じて編集すればよい`(下記へ続く)<br>

## 106. SEO SIMPLE PACKのアーカイブ非表示機能

+ wp管理画面 => `SEO PACK` => `投稿ページ` => `メディアページ` => `いいえ`を選択 => `設定を保存する`(下記へ続く)<br>

+ wp管理画面 => `メディア` => `ライブラリ` => `画像をクリック` => `添付ファイルのページを表示`(この設定で画像などの詳細のページができる)（下記へ続く)<br>

+ 上記が必要ない場合は wp管理画面 => `SEO PACK` => `投稿ページ` => `メディアページ` => `はい`を選択 => `設定を保存する`(下記へ続く)<br>

+ wp管理画面 => `メディア` => `ライブラリ` => `画像をクリック` => `添付ファイルのページを表示`(画像の個別ページはできなくなり、詳細ページがなくなり、画像がそのまま表示されるようになる)<br>

+ メディアのページを作成するか作成しないかは上記の手順で行う。<br>

+ wp管理画面 => `SEO PACK` => `タクソノミーアーカイブ` => `投稿フォーマット` => `「投稿フォーマット」のアーカイブページを使用しない` => `はい`を選択 => `設定を保存する`(現在のバージョンではあまり利用されていないので使わなければ`はい`にしておく) 余計なページが生成されなくなる<br>

+ wp管理画面 => `SEO PACK` => `その他アーカイブ` => `「著者」のアーカイブページ(著者毎のページの生成)` => `「著者」のアーカイブページを使用しない` => `はい`を選択(必要なければ) => `設定を保存する`<br>

+ インデックスをさせるさせないと言うのはGoogleの検索エンジンに辿ってもらうかどうかの選択である。

+ wp管理画面 => `SEO PACK` => `タクソノミーアーカイブ` => `カテゴリー` => `「カテゴリー」のアーカイブページをインデックスさせない` => `はい`を選択(下記へ続く)<br>

+ `タグ` => `「タグ」のアーカイブページをインデックスさせない` => `はい`を選択 => `設定を保存`(これでGoogleが辿れなくなるのでこの方が使い勝手が良くなると思われる)<br>

+ `その他アーカイブ`のところはデフォルトで`はい`が選択されているのでそのままでよいと思う<br>

+ これらの設定は状況に応じて設定すれば良いがSEOは専門的知識がないと難しい<br>
